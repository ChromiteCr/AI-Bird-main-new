## 一、我对 AI-Bird 项目的理解

AI-Bird 不是单一模型，而是一条**从快门到成片的四段式流水线**。我把每一段读到的真实代码与其职责对应如下：

| 阶段 | 真实载体 | 核心职责 | 关键产出 |
|------|----------|----------|----------|
| ① 采集端 | `camctrl.cpp`（佳能 EDSDK） | 控制相机预录、实时检测、检测到鸟自动触发录制，并把成片回传到电脑 | 原始照片/视频流 |
| ② 识别端 | `bird_classification_v2.ipynb`（ResNet50, ImageNet 预训练） | 预处理→前向推理→Softmax→Top-3 鸟种与置信度 | 每张图的 Top-3 标签 + 置信度 |
| ③ 优选端 | `README` 描述的美学评分算法 | 色块提取与融合（HSV 泛洪 + DeepLabV3 语义）→三维度量化打分→加权综合 | 0-100 美学分 + 三维分项 |
| ④ 构图端 | `image_composing.py`（BirdComposeAI） | 检测主体→生成四种构图方案（三分/黄金/对称/对角）→评分→合成与构图线叠加 | 构图建议 + 合成预览 |

### 各阶段我读到的关键细节

**① 采集端 (`camctrl.cpp`)**
- 通过 EDSDK 打开相机会话，注册 `handleObjectEvent` 回调，文件生成（`kEdsObjectEvent_DirItemCreated`）即自动下载到本地。
- `SaveTo = Both`（机内 + 电脑双存），录制槽设为 CFexpress（Slot 1）。
- 主循环靠 `EdsGetEvent()` 轮询事件。**这里的"预录/实时检测/自动触发"目前在 README 描述层，cpp 里是单次拍照 demo** —— 网页端要把它抽象成"设备状态 + 触发开关 + 回传流"，不依赖底层是否已完整实现。

**② 识别端**
- 预处理固定：Resize(256) → CenterCrop(224) → ToTensor → ImageNet 均值方差标准化 → `(1,3,224,224)`。
- `predict_bird` 多取 `top_k*2` 备用，最终返回 Top-3 的 `(label, confidence)`。
- 类别表来自 `imagenet_classes.txt`（1000 类，约 60 种鸟），含国内镜像回退逻辑。

**③ 优选端（三维度评分，网页端的核心可视化对象）**
- **构图分**：取与全图平均 HSV 差异最大的 3 个色块作视觉锚点，算各自中心到最近三分点的欧氏距离 d，`D_max = 对角线 × 0.3`，单锚点 `100×(1 - min(1, d/D_max))`，三点取平均。
- **色彩丰富度**：面积前三色占比 (p1,p2,p3) 与理想 (0.6,0.3,0.1) 求误差平方和 error，`100 × max(0, 1-error)`。
- **色彩纯净度**：色块内部色相环形标准差 + 同类色块间环形标准差，各 50% 权重，上限 90°。
- **综合**：`0.3×构图 + 0.4×丰富度 + 0.3×纯净度`，**权重用户可调** —— 这是网页端必须暴露的交互点。
- 色块融合时传统色块与 AI 语义区域色相差 > 10° 会**弹窗让用户二选一** —— 这也是一个天然的网页交互点。

**④ 构图端 (`image_composing.py`)**
- `detect_bird` 用透明通道或自适应阈值找最大轮廓得 bbox 与中心。
- 四种构图各算候选点与分数：三分法（九宫格交叉点）、黄金分割（1.618 四点）、对称（垂直/水平/中心）、对角线。
- `optimize_visual_tension` 把主体目标占比拉向 1/5，并做边缘留白约束。
- 输出三件套：合成图 / 构图线图 / `info.json`（方法、位置、缩放、分数）。

### 一句话总结
> AI-Bird 把"拍到 → 认出 → 选好 → 摆对"四件事自动化。**网页端要做的，是给这条本来在命令行/notebook 里跑的流水线，套一个能看、能调、能选的可视化驾驶舱。**

## 二、网页端定位与设计目标

**定位**：一个"可视化操纵台（Cockpit）"，不是官网。用户在这里实时看采集状态、批量浏览成片、查看 AI 识别与美学评分、手动调权重与做色块抉择、预览构图方案。

**三条设计主线**
1. **看得见**：四段流水线的状态与产出全程可视，评分不再是一个冷冰冰的数字，而是可拆解、可对比的图形。
2. **调得动**：评分权重、构图方案、色块抉择都能即时交互，结果实时重算/重排。
3. **选得快**：解决 README 反复强调的痛点——"海量相似照片不会选"，提供评分排序、阈值过滤、批量优选。

**风格基调**：科技极简、浅色（白底 + 一条主强调色 + 中性灰阶），大量留白，弱边框、轻阴影、克制动效。**预留你自己的 UI PNG 接入位**（见第六节）。

## 五、组件清单与数据流

### 可复用组件
- `StatusDot` 流水线/设备状态灯（在线/处理中/离线 三态）
- `ScoreRing` 综合分环形进度
- `ScoreBar` 三维度分项条
- `WeightSlider` 权重滑杆（联动归一化，三者和约束为 1）
- `PhotoCanvas` 照片 + 可切换图层叠加（色块/三分线/锚点/语义）
- `TopKBar` Top-3 识别置信度条形
- `ThumbCard` 图库缩略图（分数角标 + 选中态）
- `ChoiceDialog` 色块抉择弹窗（传统 vs AI）
- `SchemeRadio` 构图方案单选 + 分数

### 前端数据模型（单文件 mock，后端就位后替换为 fetch）
```js
// 一张照片在系统里的完整记录
{
  id: "IMG_0421",
  src: "assets/sample/0421.jpg",
  recognize: { topk: [["bald eagle", 0.91], ["kite", 0.05], ["vulture", 0.02]] },
  aesthetic: {
    composition: 78, richness: 88, purity: 75,
    weights: { composition: 0.3, richness: 0.4, purity: 0.3 },
    overall: 82,                      // 由 weights 实时算出
    anchors: [{x:0.33,y:0.66,hsvDiff:42}, ...],   // 归一化坐标
    blocks: [{id:3, hueDiff:12, needChoice:true, src:"trad"}, ...]
  },
  compose: {
    schemes: {
      thirds:   { score:0.82, pos:[0.33,0.66], scale:1.1 },
      golden:   { score:0.79, pos:[0.38,0.62], scale:1.0 },
      symmetry: { score:0.71, pos:[0.5,0.33],  scale:1.0 },
      diagonal: { score:0.68, pos:[0.4,0.4],   scale:1.0 }
    },
    selected: "thirds"
  }
}
```

### 关键交互的数据流
1. **权重重算**：`WeightSlider` 改值 → 归一化三权重 → `overall = w_c·comp + w_r·rich + w_p·pur` → `ScoreRing` 与图库角标全部即时刷新、图库重排序。
2. **色块抉择**：`blocks[i].needChoice` → 弹 `ChoiceDialog` → 用户选 trad/ai → 标记 `resolved` → （真实后端时触发纯净度/丰富度重算）。
3. **构图切换**：`SchemeRadio` 改 `selected` → `PhotoCanvas` 重绘对应构图线与主体位置。
4. **批量优选**：阈值滑杆定 `overall ≥ T` → 高亮/筛出达标图 → 一键导出选中集。

### 与真实后端的对接边界（饭后原型先全 mock）
| 网页动作 | 未来后端接口（FastAPI 设想） | 现有脚本 |
|----------|------------------------------|----------|

## 六、你的 UI PNG 接入方案（重要）

你提到有**自己设计、尚未完工的 UI PNG**。为不浪费你的设计，我在原型里这样预留：

- **切图位**：Logo、按钮、图标、背景纹理等都用具名容器占位（如 `<div class="slot-logo">`），并在 CSS 里用 CSS 变量 `--asset-logo: url(...)` 引用。你 PNG 一好，只改变量路径即可替换，不动结构。
- **资源目录约定**：`assets/ui/`（你的切图）、`assets/sample/`（示例照片）。
- **风格 token 先行**：颜色、圆角、阴影、间距全部抽成 CSS 变量（`:root`），你的 PNG 定稿后我据其取色微调 token，保证页面与你的视觉语言一致。
- **降级占位**：PNG 未就位时，用纯 CSS 画的极简占位（灰块 + 描边 + 居中标签），不影响功能演示。

> 请你方便时告诉我：PNG 大概覆盖哪些部分（整套界面 / 仅图标与 Logo / 仅背景），以及切图尺寸与命名习惯，我据此定 slot 规范。

## 七、技术与交付约定

- **单文件**：`index.html` 内联 `<style>` 与 `<script>`，零构建零依赖，双击即开。如交互膨胀，再视情况拆 `app.js` / `styles.css`（仍纯静态）。
- **数据**：内置 `MOCK_DATA`（一组示例照片 + 评分），结构同第五节模型；后端就位后整体换 `fetch`。
- **绘图**：色块叠加、三分线、锚点、构图线用 `<canvas>` 在照片上叠加渲染。
- **响应式**：桌面优先（操纵台场景），窄屏三栏降级为竖向堆叠。
- **可访问性**：滑杆/单选含 `aria` 标签与键盘操作，对比度达标。
