鸟类智能识别与美学评价系统 - Web端开发需求文档

1. 项目简介
这是一个综合性的鸟类AI分析后端，输入一张鸟类图像，不仅能够完成鸟类检测、精细分割、品种识别与百科生成，还会从构图、色彩丰富度、色彩纯净度三个维度给出量化美学评分与AI改善建议，并提供“懒人模式”自动输出构图裁剪和色彩微调后的优化图片。

2. 核心功能模块与技术栈
后端核心由 Python 驱动，前端需要接入本系统或复刻其功能。

（1）基础模型与识别

核心算法：YOLOv8（目标定位）、DeepLabV3-ResNet50（像素级轮廓分割）、CLIP（零样本品种识别）。

功能：输入图片，框出鸟的位置，裁剪出鸟类特写，并提取精细边缘轮廓。

输出数据：鸟类边界框坐标（Bounding Box）、精准轮廓像素坐标、裁剪后的局部图文件。

（2）品种识别与百科生成

核心逻辑：将裁剪出的鸟类特写送入 CLIP 模型，比对预设的候选鸟类列表，识别出最高置信度的品种英文名称。

信息补充：调用 DeepSeek API（配置 Key 即可），根据英文名称自动生成包括中文名、学名、简介、栖息地、食性、保护级别的鸟类百科信息，并缓存到本地 JSON 文件（bird_info_cache.json），后续重复识别直接读取，不消耗 Token。

（3）构图与美学量化评价

构图评价：基于“三分法”锚点，采用连续距离衰减算法。不再使用固定的阶梯分，而是精确计算画面视觉重心与四个三分点的距离，平滑输出 0~100 的构图分。

色彩评价：

丰富度：基于经典的 60%（主色） / 30%（副色） / 10%（点缀色） 理想模型计算误差。

纯净度：基于同类颜色的色相环形标准差计算统一程度。

综合打分：权重为 构图 0.3 + 丰富度 0.4 + 纯净度 0.3。

（4）AI 美学赏析与建议

系统将量化得分作为 Prompt 参数，发送给 DeepSeek API。

Prompt 特色：强制大模型必须基于客观得分撰写150字左右的美学赏析，并给出1~3条能够自洽于量化得分的具体修改建议（例如：构图得分低→建议裁剪调整位置；丰富度得分低→建议增加对比色）。

（5）“懒人模式”

用户开启后，系统在跑完上述分析的基础上，额外执行两步操作：

自动构图裁剪：基于所有检测到的鸟类包围盒，向最近的三分点做不超过画幅 8% 的平移裁剪（保证保守）。
自动色彩增强：如果色彩丰富度低于 60 分，在 HSV 空间自动增加 8%~15% 的饱和度；如果综合得分低于 50 分，自动提亮 7% 的亮度。
输出图片：_optimized.jpg，即经过构图微调和色彩轻度增强的最终优化图。
3. 后端接口需求说明（供 Web 开发人员参考）
如果需要将现有 Python 单文件（bird_project.py）改造成网页版，后端需要提供以下 API 接口：

接口名称 请求方式 输入参数 返回数据（JSON格式）
/upload/analyze POST 图片文件 file、懒人模式开关 lazy_mode (bool) 识别结果、每只鸟的裁剪图URL、百科信息、美学量化分数、AI修改建议报告、结果图与优化图的URL
/check/status GET 任务ID（可选） 处理进度与状态
